### 项目一：构建书籍推荐系统
#### 概述
本学期课程中，我们已学习推荐系统中的协同过滤和社交网络分析相关知识，现需将这些概念付诸实践！我会提供一个书籍推荐数据集，供你开发书籍推荐系统。本项目主题开放，你可根据兴趣自由探索，例如实现基于物品、用户或内容的协同过滤算法。该项目的核心目标是在书籍推荐系统中实现基础推荐功能：输入书籍标题/名称，需输出5本相似书籍的标题/名称。此外，鼓励你探索更多创新推荐方法和内容，并以交互式形式（如网页应用或移动应用）呈现已实现的功能。

#### 数据集
- 网站：https://www.kaggle.com/datasets/arashnic/book-recommendation-dataset/data
- 内容：
  - 用户数据（Users）：包含用户信息，用户ID（User-ID）已匿名处理，以整数形式呈现。若有可用的人口统计学数据（地区、年龄）会一并提供，否则这些字段为NULL值。
  - 书籍数据（Books）：书籍通过国际标准书号（ISBN）标识，数据集中已移除无效ISBN。同时提供部分基于内容的信息（书籍标题、书籍作者、出版年份、出版商），这些信息来源于亚马逊网络服务（Amazon Web Services）。注：若书籍有多位作者，仅显示第一位作者。数据集中还包含书籍封面图片的三种尺寸（小、中、大）的链接（Image-URL-S、Image-URL-M、Image-URL-L），链接均指向亚马逊网站。
  - 评分数据（Ratings）：包含书籍评分信息，评分（Book-Rating）分为显式评分（1-10分，分数越高表示认可度越高）和隐式评分（0分）。

#### 实验与报告要求
请按以下步骤开展实验并撰写报告：
1. 数据集信息提取（占总分20%）
   - 展示数据集的形状
   - 统计数据集中的空值总数
   - 识别数据集中存在空值的列
   - 构建对比数据集
   - 展示数据集相关信息
   - 统计数据集中每列的唯一值数量
   - 说明数据集的数据类型
2. 数据探索与预处理（占总分20%）
   - 分析数据特征、分布情况，并检查缺失值
   - 根据数据类型进行处理，包括缺失值处理、特征编码、标准化或归一化
   - 可视化目标数据分布
3. 算法定义（占总分30%）：明确将使用的推荐算法
   - 要求：至少采用三种推荐算法，需包含：
     - 课堂上讨论过的一种算法
     - 轻量梯度提升决策树（LightGBM）：https://proceedings.neurips.cc/paper_files/paper/2017/file/6449f44a102fde848669bdd9eb6b76faPaper.pdf
     - 至少一种参考算法（需复现）
   - 参考算法：
     - LightFM：用户与物品冷启动推荐的元数据嵌入（https://arxiv.org/abs/1507.08439）
     - DCN V2：改进的深度交叉网络及大规模学习排序系统实践经验（https://arxiv.org/pdf/2008.13535）
     - DeepFM：基于因子分解机的点击率预测神经网络（https://www.ijcai.org/proceedings/2017/0239.pdf）
     - BPR：基于隐式反馈的贝叶斯个性化排序（https://arxiv.org/pdf/1205.2618）
     - DIN：深度兴趣网络（https://arxiv.org/pdf/1706.06978）
4. 书籍推荐功能实现（占总分20%）：使用已定义的多种推荐算法实现书籍推荐功能
5. 系统呈现（占总分10%）：以网页或移动应用形式呈现书籍推荐系统

### 项目二：模拟参与推荐系统挑战赛
#### 概述
如今大众接触的音乐类型日益丰富，但推荐算法在关键领域仍存在挑战。缺乏足够历史数据时，算法如何判断听众是否会喜欢一首新歌或一位新歌手？又如何为全新用户推荐歌曲？国际网络数据挖掘研讨会（WSDM）向Kaggle机器学习社区发起挑战，旨在解决这些问题，构建更优的音乐推荐系统。该数据集来自亚洲领先的音乐流媒体服务平台KKBOX，其拥有全球最全面的亚洲流行音乐库，包含超过3000万首曲目。目前该平台使用基于协同过滤的算法，结合矩阵分解和词嵌入技术构建推荐系统，但认为新的技术可能会带来更优结果。

#### 数据集
- 网站：https://www.kaggle.com/competitions/kkbox-music-recommendation-challenge/overview
- 核心特征：
  - msno：用户ID
  - song_id：歌曲ID
  - source_system_tab：触发事件的标签名称，用于对KKBOX移动应用功能进行分类（例如，“我的音乐库”标签包含本地存储操作功能，“搜索”标签包含与搜索相关的功能）
  - source_screen_name：用户看到的界面名称
  - source_type：用户在移动应用上首次播放音乐的入口（如专辑、在线播放列表、歌曲等）
  - target：目标变量，target=1表示用户首次可观测的收听事件后一个月内出现重复收听行为，target=0则表示无重复收听行为

#### 实验与报告要求
请按以下步骤开展实验并撰写报告：
1. 数据探索与预处理（占总分30%）
   - 分析数据特征、分布情况，并检查缺失值
   - 根据数据类型进行处理，包括缺失值处理、特征编码、标准化或归一化
   - 数据集拆分：可在数据集网站（https://www.kaggle.com/competitions/kkbox-music-recommendation-challenge/data)
2. 模型定义（占总分10%）：选择不同的推荐模型
   - 要求：至少采用三种推荐算法，需包含：
     - 课堂上讨论过的一种算法
     - 轻量梯度提升决策树（LightGBM）：https://proceedings.neurips.cc/paper_files/paper/2017/file/6449f44a102fde848669bdd9eb6b76faPaper.pdf
     - 至少一种参考算法（需复现）
   - 参考算法：
     - LightFM：用户与物品冷启动推荐的元数据嵌入（https://arxiv.org/abs/1507.08439）
     - DCN V2：改进的深度交叉网络及大规模学习排序系统实践经验（https://arxiv.org/pdf/2008.13535）
     - DeepFM：基于因子分解机的点击率预测神经网络（https://www.ijcai.org/proceedings/2017/0239.pdf）
     - BPR：基于隐式反馈的贝叶斯个性化排序（https://arxiv.org/pdf/1205.2618）
     - DIN：深度兴趣网络（https://arxiv.org/pdf/1706.06978）
3. 模型训练（占总分10%）：在训练集上训练每个模型，并为每个模型设置合适的参数
4. 模型评估：基于预测概率与观测目标之间的ROC曲线下面积（AUC）进行评估
5. 超参数调优（占总分10%）：探索不同超参数进行调优，选择性能最佳的模型作为最终模型
6. 模型选择（占总分10%）
   - 根据评估指标选择性能最佳的模型
   - 记录最终模型配置，包括超参数值
7. 结果解释（占总分30%）
   - 总结模型评估结果，突出所选模型的优势与不足
   - 基于分析为未来工作或改进方向提供建议
